# ダジャレ生成の詳細ガイド

## ステップ1：元ネタの音を分解する

元ネタの単語をひらがな（またはカタカナ）に変換し、音の構成要素を把握する。母音列と子音列を必ず書き出すこと。

**スクリプトによる正確な分解：** `scripts/rhyme.py` が利用可能な場合は `uv run {baseDir}/scripts/rhyme.py <元ネタ>` を実行し、正確な読み・母音列・子音列を取得すること。LLM の内部知識だけで音を分解すると間違えやすいため、スクリプトが使える環境では必ず使う。スクリプトが利用できない場合は従来通り手動で分解する。

例：`uv run scripts/rhyme.py コーヒー` の出力：
```
コーヒー → こーひー (4拍)
  母音: o-o-i-i  子音: k-·-h-·
```

## ステップ2：音が似ている言葉を大量に探す（意味の距離を意識しながら）

このステップには2つの独立した探索ルートがある。**可能なら並列に実行**し、結果をマージすること。

### ルートA：LLM の語彙で連想する

以下の変換パターンを総動員して、音の重なりがある言葉やフレーズを幅広くリストアップする。このとき、元ネタと意味的にかけ離れた分野の言葉を積極的に探すこと。

探索の切り口：
- **同音異義語**：同じ読みで意味が違う言葉
- **子音交換**：子音だけ入れ替えて別の言葉にならないか（母音列を保つ）
- **母音列一致**：同じ母音配列の全く別の言葉を探す
- **部分一致・埋め込み**：元ネタの音が別の言葉の一部に含まれるケース
- **音の付加・削除**：1〜2音の追加・省略・濁点の付け外し
- **外来語・英語との掛け合わせ**：日本語の音が英語や外来語と重なるケース
- **逆方向検索**：「この言葉と音が似ている言葉は何か」ではなく「まったく関係ない面白い言葉を先に思い浮かべて、元ネタと音が重なる部分がないか探す」

リストアップの際に自問すること：
- この掛かり相手は、元ネタと意味的にどれくらい遠いか？
- 近すぎて当たり前にならないか？
- 遠すぎて何のことか分からなくならないか？

### 音数が多い元ネタ（4音以上）への対応

「アルゴリズム」「プログラミング」のような長い言葉は、全体をまるごと別の言葉に掛けるのが難しい。全体を分解して再構成する手法（「アルコール＋リズム→アルゴリズム」など）は有効だが、これだけに頼ると全部同じパターンになる。以下の別ルートも必ず検討すること。

- **部分抽出**：全体ではなく、特徴的な一部分だけを抜き出して掛ける。「アルゴリズム」なら「アルゴ」「ゴリ」「リズム」など。短い音の塊のほうが掛け相手を見つけやすい。
- **文中溶け込み**：元ネタの音を、自然な文章の中に偶然現れたかのように溶け込ませる。文全体を声に出して読んだときに元ネタの音が浮かび上がるパターン。（例：「いかがですか」の中に「いか」が溶けているタイプ）
- **文脈起点の逆算**：元ネタが使われる場面を先に想像し、その場面で自然に出てくる別の言葉から音の接点を探る。元ネタの音そのものではなく、元ネタの「世界」から攻める。
- **略称・愛称の活用**：長い言葉に略称や愛称があるなら、そちらから攻めるのも手。

### ルートB：逆引き韻辞書で網羅検索する

2つの検索モードがある。

**完全一致検索**：`uv run {baseDir}/scripts/rhyme.py --search <元ネタ>` — 母音列が完全一致する単語を辞書（約4万語）から検索する。正確な韻を見つけたいときに使う。

**韻beddingファジー検索**：`uv run {baseDir}/scripts/rhyme.py --search-embed <元ネタ>` — 母音 n-gram ベクトルによるファジー検索。母音列が部分的に似ている語や、モーラ数が異なる「語感踏み」も発見できる。幅広く候補を探したいときに使う。

サブエージェント（Agent ツール等）が使える環境では、**この検索をサブエージェントに委譲してルートAと並列に走らせる**こと。サブエージェントへの指示例：

> `uv run {baseDir}/scripts/rhyme.py --search <元ネタ>` と `uv run {baseDir}/scripts/rhyme.py --search-embed <元ネタ>` の両方を実行し、結果から元ネタと意味的に遠い面白そうな候補を10個程度ピックアップして返してください。

サブエージェントが使えない環境では直接実行する。

### マージ

ルートAとBの候補を合わせ、重複を除いたうえでステップ3に進む。辞書にない言い回しやフレーズ（ルートA）と、LLM が思いつかなかった語彙（ルートB）を組み合わせることで、候補の幅が広がる。

## ステップ3：文脈のある一文に仕上げる

音の重なりを見つけたら、それを短く・自然に・情景が浮かぶ一文に組み上げる。

仕上げのポイント：
- 可能な限り短くする。ダジャレは短歌や俳句と同じで、短いほど切れ味が出る
- 助詞・助動詞に頼った安易な繋ぎ（「〇〇が〇〇した」パターン）だけに偏らない。助詞なしで掛かるダジャレはワンランク上
- 文として読んだときに、掛かっていない部分も含めて意味が通ること
- 聞いた人の頭に情景やシチュエーションが浮かぶこと

**自然さのリトマス試験：** 完成した一文から「ダジャレだ」という意識を外して読み返す。それでも日本語の文として自然に読めるか？ 掛かりを抜きにしてもシチュエーションとして成立するか？ ここをパスしないダジャレは、音が合っていても「無理やり作った感」が出て笑えない。

やってはいけないこと：
- 掛けるためだけに不自然な状況設定を作る（「ゴリラが図鑑を読む」など、掛からなければ絶対に言わない文）
- 英語やカタカナを脈絡なくねじ込む（掛かっている部分が日本語の会話に自然に溶け込んでいないと浮く）
- 掛かりを成立させるために冗長な前置きや説明を足す（「〇〇という状況で△△が□□したら…」のような回りくどい文）

## ステップ4：検証（サブエージェントに委譲）

仕上がったダジャレの検証は、**サブエージェントに委譲する**のが望ましい。自分で作ったものを自分で評価すると確認バイアスが働き、音のズレや不自然さを見逃しやすい。

### サブエージェントが使える場合

検証エージェントに以下を渡して起動する：
- 生成したダジャレのリスト（各ダジャレの掛かっている言葉ペアを明示）
- `scripts/rhyme.py` のパス
- `references/bad-examples.md` の参照指示

検証エージェントは各ダジャレについて以下を順番にチェックし、**合格/不合格と理由**を返す：

1. **音の照合**：`uv run {baseDir}/scripts/rhyme.py <言葉A> <言葉B>` を実行し、母音列の類似度50%未満なら不合格。子音も確認し、掛かり部分の子音が大きく異なる場合も不合格（例：コーヒー k-·-h-· とノコギリ n-k-g-r は母音列100%一致だが子音が全く違い掛からない）。
2. **意味の距離**：掛かっている2つの言葉が同じカテゴリ（食べ物同士、動物同士など）に属していないか？ 同じカテゴリなら不合格。
3. **文の自然さ**：ダジャレ部分を隠して読んでも、日本語の文として成立するか？ 成立しないなら不合格。
4. **失敗パターン照合**：`references/bad-examples.md` の5つの失敗パターンに該当しないか確認。
5. **既存チェック**：有名すぎる既存ダジャレ（「布団が吹っ飛んだ」「アルミ缶の上にあるみかん」等）の焼き直しでないか。
6. **パターン重複**：他のダジャレと掛かりの手法が被っていないか？（同音異義語ばかり、等）

### サブエージェントが使えない場合

上記のチェックを自分で実行する。**特に音の照合は必ず `scripts/rhyme.py` を実行**し、LLM の内部判断だけで「合っている」と判定しないこと。

### 改善ループ

不合格が出て合格候補が3個未満になった場合、不合格の**理由を踏まえて**ステップ2-3に戻り追加候補を生成する。ただし改善ループは**1回まで**。それでも3個に満たなければ、少数でも質の高いものだけを出力する（質 > 数）。

## ステップ5：バリエーションのバランスを整える

3〜5個のうち、以下のタイプをバランスよく混ぜる。

### ストレート型（1〜2個）

判定基準：「聞いた瞬間、0.5秒以内に掛かりが分かる」もの。
作り方：音の変換距離が小さい（同音異義語、濁点の有無、1音の付加・削除）＋ 文が短い。意味の距離はそこそこでよい。「くだらなさ」や「ベタさ」が味になる。
例：「会議が怪奇」（濁点1つ）、「月曜はゲッ曜日」（音の分解）

### 知的シャレ型（1〜2個）

判定基準：「一瞬考えてから、あ、なるほど！となる」もの。
作り方：音の変換距離がやや大きい（子音交換、母音列一致、複合語への埋め込み）かつ/または、意味の距離が大きい。掛かりに気づいた時の「発見の快感」が笑いになる。文がやや長くてもよいが、掛かりの構造はクリアであること。
例：「遅刻か地獄の二択」（子音交換＋母音列一致で意味の距離が大きい）、「恋い焦がれて深煎り」（二重掛け）

ストレート型との見分け方：迷ったら「このダジャレ、小学生に言って一発で通じるか？」を考える。通じるならストレート型、少し説明がいるなら知的シャレ型。

### ぶっ飛び型（0〜1個）

**無理に入れなくてよい。** 上の2タイプで良いものが揃っているなら、無理にこの枠を埋めない。入れる場合は、長文で複数の掛かりを重ねるもの（例：「イワシ・タイ・ナマズって言わしたいな、まず」のような音の連鎖）、固有名詞を絡めた意外な組み合わせ（例：「アイムソーリー柳宗理」）、あるいは文脈ごとひっくり返すような発想の飛躍があるものに限る。英語は「その場面で自然に英語が出てくる状況」でのみ使う。
