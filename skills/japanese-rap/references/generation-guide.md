# 日本語ラップ リリック生成ガイド

## ステップ1：お題の解釈とコンセプト設計

ユーザーが入力したお題（単語、テーマ、状況）から、リリック全体の方向性を決める。

### やること

1. **お題の広がりを把握する**：お題が単語なら、その単語から連想される場面・感情・対立概念を書き出す。テーマや状況なら、その中で語り手が何を感じ、何を伝えたいかを定める。
2. **トーンを選ぶ**：以下から1つ選択する。迷ったらお題の性質から判断する。
   - **シリアス**：社会問題、内面の葛藤、人生観
   - **ユーモア**：日常の気づき、自虐、ツッコミ
   - **攻撃的（バトル）**：対戦相手への挑発、自分の実力誇示
   - **内省的**：過去の振り返り、自分への問いかけ
3. **小節数を決める**：ユーザーから指定がなければ **16小節**（=16行）をデフォルトとする。8小節の場合は凝縮が求められるため、パンチラインの密度を上げる。

### 構成テンプレート（16小節の場合）

| 区間 | 小節 | 役割 |
|---|---|---|
| 起 | 1〜4 | 状況設定、世界観の提示。聞き手を引き込むフック |
| 承 | 5〜8 | テーマの展開、ストーリーの深掘り |
| 転 | 9〜12 | 視点の転換、対比、盛り上げ |
| 結 | 13〜16 | 最大のパンチライン、メッセージの着地 |

8小節の場合は 起(1-2) → 承(3-4) → 転(5-6) → 結(7-8) に圧縮する。

## ステップ2：キーワードの音を分解する

韻を踏むための「種」を仕込むフェーズ。お題に関連するキーワードの音構造を正確に把握する。

### やること

1. **キーワードリストアップ**：お題と関連する言葉を5〜10個挙げる。同義語だけでなく、対義語、連想語、同じ場面に登場する言葉を幅広く。
2. **rhyme.py で音を分解する**：各キーワードに対して `uv run {baseDir}/scripts/rhyme.py <キーワード>` を実行し、読み・母音列・子音列を取得する。LLM の内部知識だけで音を判断すると間違えやすいため、**スクリプトが使える環境では必ず使う**。

```
例：uv run scripts/rhyme.py 人生
人生 → じんせい (4拍)
  母音: i-·-e-i  子音: j-N-s-·
```

```
例：uv run scripts/rhyme.py 信念 人生 覚悟
単語    読み      拍  母音        子音
信念    しんねん   4  i-·-e-·    sh-N-n-N
人生    じんせい   4  i-·-e-i    j-N-s-·
覚悟    かくご     3  a-u-o      k-k-g
```

3. **韻の種を選定する**：母音列が近いキーワード同士をマークし、韻ペア探索の出発点とする。特に小節末に配置したい重要ワードを決める。

## ステップ3：韻のペアを大量に探す（2ルート並列）

2つの独立した探索ルートを**可能なら並列に実行**し、結果をマージする。

### ルートA：LLM の語彙で連想する

ステップ2で選定したキーワードをもとに、音が近い言葉を幅広くリストアップする。ラップでは**意味の距離が近くても構わない**。むしろテーマに沿った言葉同士で韻を踏むと説得力が出る。

探索の切り口：
- **母音列一致**：同じ母音配列の言葉を探す（韻の基本）
- **末尾2〜3拍の母音一致**：脚韻として最も使いやすい
- **子音交換**：母音を保ったまま子音だけ変える
- **複合語の分解・再構成**：長い言葉を分解して韻の素材にする
- **外来語・スラング**：ラップでは自然に使える語彙が広い
- **固有名詞**：地名、人名、ブランド名なども韻の素材になる

### ルートB：rhyme.py --search で韻辞書検索

`uv run {baseDir}/scripts/rhyme.py --search <キーワード>` を実行し、母音列が一致・類似する単語を辞書（約4万語）から網羅的に検索する。

サブエージェント（Agent ツール等）が使える環境では、**この検索をサブエージェントに委譲してルートAと並列に走らせる**こと。サブエージェントへの指示例：

> `uv run {baseDir}/scripts/rhyme.py --search <キーワード>` を実行し、完全一致と類似パターンの結果から、テーマに関連しそうな候補・意外性のある候補を合わせて15個程度ピックアップして返してください。

### マージと選定

ルートAとBの候補を合わせ、以下の基準で韻ペアを選ぶ。16小節なら最低8ペア（各小節末の脚韻用）必要。余裕を持って12〜16ペア候補を確保する。

### 韻の「固さ」の判定基準

韻の固さ（踏めている度合い）を以下で判定する：

1. **母音列一致率**：`uv run {baseDir}/scripts/rhyme.py <言葉A> <言葉B>` で確認。**50%以上**を合格ラインとする。70%以上なら「固い韻」。
2. **子音の近さ**：母音が一致していても、子音が全く違うと韻として弱い。以下の子音は互いに近いとみなす：
   - 無声/有声ペア：k-g, t-d, s-z, h-b, p-b
   - 調音点が近い：s-sh, t-ch-ts, n-m, r-n
3. **拍数の一致**：拍数が同じペアほど韻としてきれいに聞こえる。1拍差は許容、2拍以上差がある場合はフロウで吸収する必要がある。
4. **末尾一致の優先**：日本語ラップでは脚韻（行末）が最重要。末尾2〜3拍の母音が一致していれば、先頭が多少ずれていても韻として成立する。

## ステップ4：リリック構築

韻ペアを配置し、小節を肉付けしてリリックを組み上げる。

### 基本ルール

- **1小節 ≒ 12〜18文字（日本語モーラ）を目安**とする。短すぎると情報量不足、長すぎるとフロウに乗らない。
- **韻は小節末（脚韻）に配置**するのが基本。2行1組で韻を踏む（AA BB 形式 or ABAB 形式）。
- **韻を踏むだけでなくパンチラインを仕込む**。韻がきれいでも内容が空っぽなら意味がない。

### 文字数の調整方法

目安の12〜18文字に収まらない場合：
- **長すぎる場合**：助詞を省略する、漢語を和語に（またはその逆に）置き換える、修飾語を削る
- **短すぎる場合**：修飾語を足す、具体的な描写を加える、感嘆詞やフィラーを入れる（ただしフィラーの多用は禁物）
- **全小節を同じ文字数にしない**：単調なリズムを避けるため、あえて長短をつける。等間隔のリズムは退屈になりやすい。

### パンチラインの仕込み方

パンチラインとは、聞き手に「おっ」と思わせる一行。以下の手法で仕込む：

- **ダブルミーニング**：一つの言葉が二つの意味を持つように配置する。探し方のコツ：韻ペアを決めた後、その言葉に別の意味がないか辞書的に確認する。動詞の自動詞/他動詞、名詞と動詞の同音異義、比喩的用法と字義通りの用法など。
  - 例：「道を切り開く」→ 物理的に道を切る + 人生を切り開く
  - 例：「熱い」→ 温度が高い + 情熱がある
- **比喩（メタファー）**：抽象的な概念を具体的なモノに喩える。お題のテーマと全く違う領域から借りてくると鮮度が出る。
- **対比・反転**：前の小節で言ったことを後の小節でひっくり返す。期待を裏切る展開。
- **固有名詞の引用**：有名人、地名、作品名を文脈に合わせて引用し、意味を重ねる。

### フロウの記法（テキストベースでの表現）

テキストでリリックを提示する際、フロウ（リズム・区切り）を示すために以下の記法を使う：

| 記号 | 意味 | 例 |
|---|---|---|
| `/` | 小節の区切り（行末） | 俺の言葉が街を照らす / |
| `_` | 休符（間を置く箇所） | 立ち止まって _ 振り返る / |
| `**太字**` | パンチライン・強調 | 夜明け前が **一番暗い** / |
| `[韻]` | 韻を踏んでいる箇所の注記 | 信念[i-e-N] 貫く人生[i-e-i] / |

### 構成の組み立て方

1. まずステップ1で決めた起承転結テンプレートに沿って、各区間で言いたいことをメモ書きする。
2. 韻ペアを各区間に割り振る。結（最後の4小節）に最も固い韻・最も強いパンチラインを配置する。
3. 各小節を肉付けする。韻の位置（行末）を先に固定し、そこに向かって前半を書く。
4. 全体を通して読み、フロウが単調になっていないか確認する。

## ステップ5：検証（サブエージェントに委譲）

完成したリリックの検証は、**サブエージェントに委譲する**のが望ましい。自分で作ったものを自分で評価すると確認バイアスが働き、韻のズレやフロウの問題を見逃しやすい。

### サブエージェントが使える場合

検証エージェントに以下を渡して起動する：
- 生成したリリック全文
- 各韻ペアの一覧（どの行とどの行で踏んでいるか）
- `scripts/rhyme.py` のパス
- `references/bad-examples.md` の参照指示（存在する場合）

### チェック項目

検証エージェントは以下を順番にチェックし、**合格/不合格と理由**を返す：

1. **韻の照合**：`uv run {baseDir}/scripts/rhyme.py <言葉A> <言葉B>` を各韻ペアに対して実行し、母音列の類似度を確認する。50%未満なら不合格。子音も確認し、子音が大きく異なる場合は要注意として報告する。
2. **フロウチェック**：各小節の文字数（モーラ数）を数え、以下を確認する。
   - 12〜18文字の目安から大きく外れている小節はないか（±4文字以上は警告）
   - 全小節が同じ文字数で単調になっていないか
   - リズムの変化（長短の組み合わせ）があるか
3. **パンチラインチェック**：16小節中、最低2箇所以上のパンチライン（ダブルミーニング、比喩、対比のいずれか）が含まれているか。パンチラインが0箇所なら不合格。
4. **内容の一貫性**：起承転結の流れが破綻していないか。唐突なテーマ転換がないか。
5. **失敗パターン照合**：`bad-examples.md` が存在する場合、記載された失敗パターンに該当しないか確認する。
6. **韻パターンの多様性**：同じ母音列の韻ばかり使っていないか。最低2種類以上の母音列パターンで韻を踏んでいること。

### サブエージェントが使えない場合

上記のチェックを自分で実行する。**特に韻の照合は必ず `scripts/rhyme.py` を実行**し、LLM の内部判断だけで「踏めている」と判定しないこと。

### 改善ループ

不合格項目が出た場合はステップ6（推敲）に進んで修正する。

## ステップ6：推敲

検証で指摘された箇所を修正し、リリック全体を仕上げる。

### やること

1. **韻の修正**：不合格になった韻ペアを差し替える。ステップ3の候補リストに戻り、別のペアを選ぶ。
2. **フロウの調整**：文字数が偏っている小節を書き直す。長すぎる行は分割、短すぎる行は肉付け。
3. **パンチラインの強化**：パンチライン不足の場合、既存の小節にダブルミーニングを仕込めないか再検討する。
4. **通し読み**：全小節を頭から通して読み、以下を最終確認する。
   - 起承転結の流れが自然か
   - テーマから逸脱していないか
   - 同じ言い回し・表現が重複していないか
   - 最後の2小節が最も印象に残るか（締めが弱くないか）
5. **再検証**：修正が大きい場合は、再度ステップ5の検証を通す。ただし推敲ループは**2回まで**とし、それ以上は現状のベストで出力する。
